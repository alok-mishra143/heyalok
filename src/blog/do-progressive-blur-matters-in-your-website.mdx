<p id="blog-heading-0-el-0">So hello guys, are you tired of trying backdrop-blur-sm, backdrop-blur-xl, etc., and still not satisfied with the blur? Sometimes it looks awful.</p><p id="blog-heading-0-el-1">Let me give you an example of two types of blur:</p><ol id="blog-heading-0-el-2"><li id="blog-heading-0-el-3"><strong>Normal backdrop blur</strong></li></ol><figure id="blog-heading-0-el-4"><img src="https://cdn-images-1.medium.com/max/807/1*xW61Ytw2uhrKnJDCeNdqDA.png" loading="lazy" /></figure><p id="blog-heading-0-el-6">Here you can clearly see an edge. There is a visible line that separates the blurred area.</p><p id="blog-heading-0-el-7">2. <strong>Progressive blur</strong></p><figure id="blog-heading-0-el-8"><img src="https://cdn-images-1.medium.com/max/1024/1*tAKJUPeTyhyHf6ANXS3RGg.png" loading="lazy" /></figure><p id="blog-heading-0-el-10">In the second picture, you can see that the separating line is gone. The blur looks much more professional while still keeping the header distinct from the rest of the content.</p><p id="blog-heading-0-el-11">So the question is: how can you build this?</p><p id="blog-heading-0-el-12">There are two ways to build this. I’ll show you from the easiest to the hardest.</p><p id="blog-heading-0-el-13">→ First, using a gradient mask image.</p><p id="blog-heading-0-el-14">You can create a gradually blurred SVG in Figma and use it directly in your project. The downside is that it may not produce the exact result you want. You’ll also have limited customization over the blur and how you apply it in your project.</p><p id="blog-heading-0-el-15">→ Using pure CSS with some math.</p><p id="blog-heading-0-el-16">You can look at examples like <a href="https://magicui.design/docs/components/progressive-blur">MagicUI</a> or <a href="https://reactbits.dev/animations/gradual-blur">ReactBits</a>. I found ReactBits to be complex, with external dependencies and heavy math.</p><p id="blog-heading-0-el-17">Then I found a simpler <a href="https://codepen.io/silas/pen/rNYqZoz">CodePen</a> example that was easier to understand.</p><p id="blog-heading-0-el-18">After spending an hour studying it, I was finally able to build my own progressive dynamic blur component with no external libraries and much simpler logic.</p><p id="blog-heading-0-el-19">progressiveblur.tsx <a href="https://ray.so/9UNs5WB">code Link</a></p><p id="blog-heading-0-el-20"><strong>How the Progressive Blur Work</strong></p><figure id="blog-heading-0-el-21"><img src="https://cdn-images-1.medium.com/max/1024/1*aGTH5pHfvrgDPpiRtXc1mw.png" loading="lazy" /></figure><p id="blog-heading-0-el-23">This picture shows how a progressive blur works.</p><p id="blog-heading-0-el-24">Instead of applying the same blur everywhere, the blur is divided into multiple horizontal layers. Each layer has a different intensity.</p><p id="blog-heading-0-el-25">At the bottom, the blur is very light. As you move upward, the blur slowly increases. By the time you reach the top, the blur is much stronger.</p><p id="blog-heading-0-el-26">The idea is simple: stack several sections on top of each other and gradually increase the blur value. This creates a smooth transition. It removes the harsh visible line you usually get with a single backdrop blur and makes the header feel naturally separated from the rest of the content</p><p id="blog-heading-0-el-27"><strong>Where does progressive blur shine?</strong></p><ul id="blog-heading-0-el-28"><li id="blog-heading-0-el-29">It works especially well in card components where you want to show text on top of an image. The gradual blur keeps the image visible while making the text readable.</li><li id="blog-heading-0-el-30">It’s also great for headers and footers. It helps separate sections without using harsh borders or solid backgrounds.</li></ul><p id="blog-heading-0-el-31">This type of blur adds a clean, premium feel to the UI and improves the overall user experience.</p><p id="blog-heading-0-el-32"><strong><em>Some Example of Progressive from Internet</em></strong></p><figure id="blog-heading-0-el-33"><img src="https://cdn-images-1.medium.com/max/1024/0*-KNPOoX0Fvmg0mmx" loading="lazy" /></figure><p id="blog-heading-0-el-35">Some Portfolio that uses Progressive blur</p><ol id="blog-heading-0-el-36"><li id="blog-heading-0-el-37"><a href="https://zerion0.site/">Zerion</a></li><li id="blog-heading-0-el-38"><a href="https://legions.dev/">legion dev</a></li></ol>
